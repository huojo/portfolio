Description of the project

The app was made with Typescript React, using Firebase as a database. The app employs React router and Firebase authentication. When the user first arrives to the app, they will see a login page, from which they can navigate to the sign up page or enter their credentials and login and be directed to the home page.

Functionalities

  • User authentication with Firebase (login/logout)
  • Email and password type sign up with Firebase
  • Navigation with React router
  • Different users only see their own shopping list items
  • The user can add a new item for their list
  • The user can edit an item on their list (Firebase setDoc)
  • The user can remove an item from their list
  • The user can mark an item as checked or unchecked and UI shows the difference (Firebase update)
  • The total price of all the items is listed below
  • The items are separated under each store, so the user can easily tell which items to get from S-Market and which from K-Market or Lidl

Structure of the app

The App file handles the router, which directs the user to Home, Login or Sign up pages based on where they need to go. Every page shows the Banner to the user, whether they are logged in or not. Once the user is logged in, App directs the user to Home, which has inside it the ShoppingList. The ShoppingList keeps within it:

  • AddForm for adding new items to the user's shopping list
  • ShoppingList maps the items useState array and forms a list of ListItemDefault components, which show either the specific item's information, or the editing form so the user can edit the item. Inside ListItemDefault file there is also the EditDropDown component that ListItemDefault calls if the user is editing the item so the remaining options can be dynamically created

Grade proposal, Achievements, Improvements

  • Grade proposal: I propose full marks on this (grade 2) because the functionalities the teacher and I agreed on at the beginning work

    • Firebase update AND setDoc (versatile use of Firebase)
    • Use of models and types (Typescript)
  • Improvements, eg. "If I had unlimited time for this"
    • A user could have multiple shopping lists, and then use previous shopping lists as the basis for their new shopping lists
    • Separate pages for those shopping lists (further into React Router)
    • Typescript is still new to me, and I don't think I took full advantage of it. If I knew more about Typescript, I could remove "| undefined | null" from certain types. I would've rather not used those at all, but the app wouldn't work without them.
    • Currently the app renders quite a few times just upon entering the pages, so that could be improved upon, I think
    • It would've been better to add redux toolkit to this, so I wouldn't have to pass props so much. At one point, there were like 16 props passed to one component, although I ultimately refactored the code to work better, but for example the user and email are passed from one child to the parent's grandchild, which is not ideal.
